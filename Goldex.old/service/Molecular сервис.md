**Molecular сервис:**

Создай полностью автономный микросервис на **TypeScript**, совместимый с фреймворком **Moleculer**, предназначенный для обработки **покупок и продаж драгоценных металлов через вендинг-устройства** (например, золотых слитков или монет).  
Требования:
1. **Функциональность**:
   - Два основных действия (actions):
     - `purchase`: принимает тип металла (например, `"gold"`, `"silver"`), массу в граммах и способ оплаты; возвращает итоговую стоимость и статус транзакции.
     - `sell`: принимает тип металла, массу и данные о клиенте; возвращает расчётную выкупную сумму и подтверждение приёма.
   - Валидация входных данных с использованием `fastest-validator`.
   - Имитация взаимодействия с внешними системами (например, курсами металлов или платёжным шлюзом) через моки или простые логические правила (например, фиксированный курс + наценка).
2. **Архитектура**:
   - Сервис должен запускаться в **standalone-режиме**: использовать встроенный брокер Moleculer, локальный транспорт (`NATS` не требуется), `MemoryAdapter` для внутреннего состояния (если необходимо).
   - Чистый раздел на `service.ts` (или `src/services/metal-vending.service.ts`) и `moleculer.config.ts`.
3. **Инфраструктура**:
   - Включить `Dockerfile` (на базе `node:20-alpine`) с multi-stage сборкой TypeScript.
   - Добавить `docker-compose.yml`, запускающий сервис в изоляции (без внешних зависимостей).
   - Добавить `.gitlab-ci.yml` с этапами: `lint`, `build`, `test` (можно заглушки) и `build-image`.
4. **Документация и сопровождение**:
   - Краткий `README.md` с инструкцией по локальному запуску через `docker-compose up --build`.
   - JSDoc/TSdoc для всех публичных методов и интерфейсов.
   - `.env.example` (если используются переменные окружения, например, `METAL_API_MOCK=true`).
Сервис должен быть готов к немедленному запуску и демонстрации базового workflow вендинга драгоценных металлов.
